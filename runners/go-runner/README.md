# OpenContest Go Runner

Высокопроизводительный ранер для проверки задач по программированию. Поддерживает динамическое добавление языков через систему "модов" (JSON-конфигурации).

## Как это работает

Ранер спроектирован как модульная платформа. Все настройки языков вынесены в отдельные JSON-файлы. При сборке Docker-образа система автоматически сканирует эти файлы и устанавливает необходимые компиляторы и интерпретаторы.

## Добавление нового языка

Чтобы добавить поддержку нового языка, вам **не нужно** менять исходный код ранера или `Dockerfile`. Достаточно выполнить два шага:

### Шаг 1: Создайте JSON-конфигурацию
Создайте файл в директории `languages/` (например, `php.json`):

```json
{
  "name": "php",
  "packages": ["php-cli"],
  "source_file": "main.php",
  "run_command": ["/usr/bin/php", "main.php"]
}
```

### Шаг 2: Соберите Docker-образ
Просто запустите сборку образа. Скрипт автоматизации сам найдет новый пакет `php-cli` и установит его в образ.

```bash
docker build -t opencontest-runner .
```

## Структура JSON-файла

| Поле | Описание |
| :--- | :--- |
| `name` | Уникальный ID языка (используется в задачах). |
| `packages` | Список системных пакетов (apt), необходимых для этого языка. |
| `source_file` | Имя файла, в который будет сохранен исходный код (например, `Main.java`). |
| `compile_commands` | (Опционально) Список команд для компиляции. Каждая команда — это массив строк. |
| `run_command` | Команда для запуска решения. |

### Пример сложной конфигурации (C++)
```json
{
  "name": "cpp",
  "packages": ["g++"],
  "source_file": "main.cpp",
  "compile_commands": [
    ["g++", "-std=c++17", "-O2", "main.cpp", "-o", "main"]
  ],
  "run_command": ["./main"]
}
```

## Архитектура проекта

- `main.go`: Точка входа, работа с RabbitMQ и пульс.
- `core/languages.go`: Динамическая загрузка JSON-конфигураций.
- `core/judge.go`: Логика изоляции (NsJail), компиляции и запуска тестов.
- `core/state.go`: Управление внутренним состоянием ранера.
- `languages/`: Директория с "модами" языков.
- `setup_env.sh`: Скрипт автоматической установки зависимостей при сборке.

## Изоляция
Для безопасного запуска кода используется **NsJail**. Он ограничивает доступ к файловой системе, сети и ресурсам (CPU, RAM). Конфигурация NsJail задается через переменные окружения в `Dockerfile`.
