OpenContest

Платформа для онлайн-олимпиад по спортивному программированию: участники решают задачи, backend публикует проверки в RabbitMQ, runner-ы запускают код в sandbox через nsjail, а админка показывает состояние системы.

Стек: React/Vite, Django REST Framework, PostgreSQL, RabbitMQ, Go runner, Docker Compose, Kubernetes и KEDA.

React Django PostgreSQL RabbitMQ Go nsjail Kubernetes KEDA

Быстрый старт

На Windows рекомендуется запускать из PowerShell:

.\scripts\opencontest.ps1 start

После запуска доступны:

  • Frontend: http://localhost:5173
  • Backend API: http://localhost:8000/api
  • Backend health: http://localhost:8000/api/health/
  • RabbitMQ UI: http://localhost:15672, логин guest/guest

Демо-пользователи:

  • admin: admin / admin123
  • participant: coder / coder123

Архитектура

Frontend

React/Vite приложение для участников и админов. Production image отдается через nginx.

Backend

Django REST API хранит контесты, задачи, посылки, standings и состояние runner-ов.

Queue

RabbitMQ хранит задания проверки в очереди judge.submissions.

Runner

Go-сервис компилирует и запускает код через nsjail, затем отправляет verdict обратно.

Participant -> Frontend -> Backend -> RabbitMQ -> Runner -> nsjail
                                    ^                         |
                                    |                         v
                                    +------ result callback ---+

Runner и nsjail

Runner поддерживает C++17, Python 3, Java 17 и Rust. Он отправляет heartbeat в backend, поэтому frontend может получать актуальный список языков.

Runtime image включает компиляторы, интерпретаторы и бинарь nsjail. В локальном Docker Compose runner запускается с privileged: true и seccomp=unconfined, потому что namespace/cgroup операции часто блокируются дефолтным профилем Docker.

Для production нужно закрепить ревизию NSJAIL_REF, настроить securityContext и выделить отдельный node pool под runner-ы.

Админка

  • создание и редактирование задач;
  • создание контестов;
  • просмотр всех посылок;
  • rejudge посылки;
  • ручной override verdict-а;
  • disqualify пользователя в рамках контеста;
  • live-статистика runner-ов: online, busy, queue, last verdict, jobs processed.

Kubernetes

Манифесты лежат в deploy/k8s. Главная команда:

.\scripts\opencontest.ps1 k8s-start

KEDA ScaledObject масштабирует runner deployment по длине RabbitMQ queue.

kubectl -n opencontest get hpa
kubectl -n opencontest get pods -l app=runner

Нагрузочный тест

Скрипт создает много посылок, чтобы увидеть, как очередь растет и как KEDA добавляет runner pod-ы.

$env:OPENCONTEST_LOAD_COUNT='80'
$env:OPENCONTEST_LOAD_SLEEP='15'
$env:OPENCONTEST_LOAD_WORKERS='10'
.\backend\.venv\Scripts\python.exe scripts\load_test.py

Roadmap

Уже реализовано

  • локальный запуск через Docker Compose;
  • авторизация и роли;
  • API контестов, задач, посылок и standings;
  • RabbitMQ queue и Go runner;
  • nsjail sandbox;
  • динамический список языков из runner heartbeat;
  • live-статистика runner-ов в админке;
  • rejudge, override verdict-а и disqualify;
  • Prometheus-style metrics endpoint;
  • Kubernetes manifests и KEDA autoscaling.

Планируется

  • полноценный импорт problem archives;
  • больше вариантов custom checkers;
  • backend/runner/frontend тесты;
  • CI/CD;
  • production secrets и image registry;
  • code splitting frontend bundle;
  • более строгая изоляция runner-ов.
